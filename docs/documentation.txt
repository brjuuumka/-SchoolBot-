SCHOOLBOT - ТЕХНИЧЕСКАЯ ДОКУМЕНТАЦИЯ

ОПИСАНИЕ ПРОЕКТА
----------------
SchoolBot — это Telegram-бот, предназначенный для автоматизации школьного документооборота. Бот заменяет бумажный дневник, позволяя ученикам, родителям и учителям взаимодействовать через удобный интерфейс Telegram.

ТЕХНОЛОГИЧЕСКИЙ СТЭК
--------------------
- Язык: Python 3.x
- Библиотека: python-telegram-bot (версия 20.x)
- База данных: SQLite 3
- Аутентификация: хэширование паролей (hashlib)

АРХИТЕКТУРА ПРОЕКТА
-------------------
Проект организован следующим образом:

bot/
├── main.py           # Точка входа, инициализация бота и маршрутизация
├── database.py       # Работа с SQLite (все CRUD-операции)
├── auth.py           # Аутентификация и управление сессиями
├── admin.py          # Функционал администратора
├── teacher.py        # Функционал учителя
├── student.py        # Функционал ученика
├── parent.py         # Функционал родителя
├── constants.py      # Токен бота, состояния и константы
└── schedule.json     # Файл с расписанием для тестового класса

ОСНОВНЫЕ КОМПОНЕНТЫ
-------------------
1. База данных (Database)
   - Класс Database в database.py управляет подключением к SQLite
   - Использует WAL-режим и блокировки для конкурентного доступа
   - Содержит методы для всех операций с таблицами

2. Аутентификация (AuthHandler)
   - Управляет входом пользователей по логину и паролю
   - Хранит активные сессии в user_sessions
   - Сохраняет telegram_id после первого входа

3. Обработчики по ролям
   - AdminHandler: статистика, регистрация пользователей, рассылки
   - TeacherHandler: работа с классами, выставление оценок, ДЗ
   - StudentHandler: просмотр оценок, расписания, замечаний
   - ParentHandler: отслеживание успеваемости детей
   - ClassTeacherHandler: дополнительные функции (рассылки классу)

4. Маршрутизация сообщений
   - Обработка команд: /start, /cancel
   - ConversationHandler для многошаговых операций (регистрация, рассылка)
   - CallbackQueryHandler для инлайн-кнопок

СТРУКТУРА БАЗЫ ДАННЫХ
---------------------
Таблицы (см. schema.sql):
- users — пользователи (ученики, родители, учителя, админы)
- classes — классы и классные руководители
- student_parents — связь учеников с родителями
- grades — оценки учеников
- attendance — посещаемость
- homeworks — домашние задания
- comments — замечания от учителей
- notifications — уведомления
- schedule — расписание уроков

НАСТРОЙКА И ЗАПУСК
------------------
1. Установите зависимости: pip install -r requirements.txt
2. Настройте токен бота в bot/constants.py
3. Запустите бота: python bot/main.py

ПРИМЕЧАНИЯ
----------
- Бот использует SQLite, поэтому дополнительный сервер БД не требуется
- Все пароли хранятся в хэшированном виде
- Уведомления отправляются асинхронно через Telegram Bot API
- Файл schedule.json используется только для первоначального заполнения расписания
